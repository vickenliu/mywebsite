What 'scope' is, and how it works in javascript

'Scope' from the word we know it defines a certain area or demension if you call it.
In Javascript, the main idea of scope is that we used to claim varaible or method's activity area, those defined varaibles or methods are only valid in their according scope. Computer can not retrieve them out of their scope.
Like laws as varaibles, we have state-grade laws and province-grade laws, even towns-grade laws/rules. the laws defined state-grade can be reached by all of children-scopes defined within the country. But from the country-level, we can not get or use the province/town level laws. because those children-scope leveled laws only available within their own scope.same rules for province and towns's relationship. provincal laws are available to towns,but towns laws are only reachable within town. and so on so forth.
Another very important rule is, if there are the same named laws with in the state and children-level area, the children-scope law will effect to the local area not the state or parent-level's law. which is propabely totally different with the real reality. But in javascript that is how it works.
In javascript,if we say whole js file as a country. we have global scope which is like the whole country, and then we may have lots of defined functions within which we also can define its own variables , they are only availbe within the function scope.out of the function,we can't retieve them. Variables defined in a function belongs to their direct parent function's scope.
In javascript, we only have 2 ways of scopes, global area, and function-area scope.There is no block level scope within this version's js.
